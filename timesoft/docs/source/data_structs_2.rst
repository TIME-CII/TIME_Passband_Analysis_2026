Level 2: Maps and Linemaps
==========================

The **level 2** data products of TIME observations
are maps and, for one-dimensional scan patterns,
"line maps" of data binned along the scan axis.
These data can be generated by the :doc:`timesoft.Timestream <timestream>` 
class and, like the **level 1** timestream data are
saved in zipped numpy file archives (``.npz``).
They can be loaded, analyzed, and saved using 
methods of the ``timesoft.Map`` class (2D data)
and ``timesoft.LineMap`` class (1D data).

Contents of the Map and LineMap Files
-------------------------------------

The map ``.npz`` files contain a number of elements.

====================    ==========   ======================================================================
Data Element            Data Type    Description
====================    ==========   ======================================================================
header                  dictionary   Summary information about the file/observation
x_edge                  1D array     The edges of map pixels along the x-axis
y_edge                  1D array     The edges of map pixels along the y-axis
maps                    3D array     An n_detectors x map_y_size x map_x_size array of maps
counts                  2D array     The number of timestream samples averaged in each pixel
extras                  dictionary   A dictionary containing additional 2D arrays
====================    ==========   ======================================================================

The line map ``.npz`` files contain a similar elements, but with reduced dimensions

====================    ==========   ======================================================================
Data Element            Data Type    Description
====================    ==========   ======================================================================
header                  dictionary   Summary information about the file/observation
ax_edge                 1D array     The edges of the bins along the scan axis
linemaps                2D array     An n_detectors x n_bins array of binned data
counts                  1D array     The number of timestream samples averaged in each bin
extras                  dictionary   A dictionary containing additional 1D arrays
====================    ==========   ======================================================================

Headers
-------

Most items in the headers of **level 1**, 
**level 2** and **level 3** data (timestreams,
maps, and power spectra) are the same,
and simply inherited from the lower level data
products. See the section on contents of
the :ref:`timestream headers <header_ts>` 
for information on these basic contents.
  
Any given file may also have 
arbitrary items added to the header (which 
will in turn be inherited by derived data
products). These can be used for exptending
the functionality of the data analysis pipeline,
or simply for adding notes to a given piece of 
data.

Maps and LineMaps each have one unique piece of 
header information that is not associated with 
their parent timestream.

**Additional Map Header Entry 'map_pars':** 
This header key references another dictionary containing
information about the ``timesoft.Map`` object. It 
includes pixel size, map center, and map dimensions.

**Additional LineMap Header Entry '1d_pars':** 
This header key references another dictionary containing
information about the ``timesoft.LineMap`` object. It 
includes bin size, scan center, and scan length.

The 'Extras' Attribute
----------------------

The ``.extras`` attribute of a ``timesoft.Map`` or 
``timesoft.LineMap`` instance is a dictionary, which can
be used to store additional data, binned on the same grid
as the map or linemap data. This can be used to things like 
the average elevation of samples falling in given pixel/bin.

Maps of arbitrary properties can be stored in the extras 
dictionary, and should be assoicated with a descriptive key.

.. note::

    This feature is currently only implemented for the
    ``timesoft.LineMap`` class.
